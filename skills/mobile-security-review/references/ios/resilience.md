# iOS — MASVS-RESILIENCE (static)

Scope: static review of an iOS artifact (`.ipa` / `.app` / Mach-O) for the presence and basic robustness of anti-reverse-engineering / anti-tampering defenses — jailbreak detection, anti-debugging, anti-hook/Frida detection, emulator/virtualization detection, file-integrity & code-signature self-checks, `get-task-allow`/debuggable on release, and obfuscation — read from the binary, `Info.plist`, and the embedded `entitlements.plist`/code-signature without a device. **Assess only under the R profile** — under L1/L2 the *absence* of these controls is not a finding (see Classic false positives).

Anchor each finding to the exact control in SKILL.md (read it); verify MASWE/MASTG IDs against the live guide.

> Everything here is a **candidate generator** — promote to a finding only after the anti-false-positive gate in `SKILL.md`. RESILIENCE is **defense-in-depth and never a substitute** for the L1/L2 controls (STORAGE/CRYPTO/NETWORK/PLATFORM/CODE); a missing resilience control does not excuse, say, a plaintext secret. **Static reach is limited:** statically you can find the *references* (symbols, strings, syscalls) that implement a defense, but you **cannot judge whether it actually fires, is wired into a security-relevant path, or survives a one-line bypass** — that is the runtime test (`ios-dynamic.md`). Report static RESILIENCE results as *presence/absence of the mechanism*, mark effectiveness **Needs dynamic verification**, and treat single-point checks as trivially bypassable. **FairPlay caveat:** if the binary is encrypted (`cryptid=1`), `otool`/`nm`/`strings`/Ghidra read **ciphertext** for `__TEXT` — every symbol/string-based detection below is **invalid until decrypted**, so "no jailbreak/anti-debug refs" on an encrypted binary is not a finding (see `ios-overview.md` Tools).

## Group MASVS controls (verbatim, MASVS v2.0.0)
- **MASVS-RESILIENCE-1** — The app validates the integrity of the platform.
- **MASVS-RESILIENCE-2** — The app implements anti-tampering mechanisms.
- **MASVS-RESILIENCE-3** — The app implements anti-static analysis mechanisms.
- **MASVS-RESILIENCE-4** — The app implements anti-dynamic analysis techniques.

Anchor by *statement*, not by group. RESILIENCE numbering splits by **what the defense resists**: platform-integrity / jailbreak → **-1**; tamper/repackaging/integrity self-check → **-2**; static-analysis resistance (obfuscation, symbol stripping) → **-3**; dynamic-analysis resistance (anti-debug, anti-hook, emulator, the debuggable/`get-task-allow` flag) → **-4**. Common mis-anchors: jailbreak detection is **-1** (not -4); `get-task-allow`/debuggable is **-4** (not CODE, not -2); symbol stripping is **-3** (not -2). And remember these only count under **R**.

## Checks

### RESILIENCE-1 — platform-integrity / jailbreak detection
- **Jailbreak-detection references present at all (static map)** — grep the (decrypted) binary for the canonical checks: filesystem probes (`/Applications/Cydia.app`, `/Applications/Sileo.app`, `/Applications/Zebra.app`, `/bin/bash`, `/bin/sh`, `/usr/sbin/sshd`, `/etc/apt`, `/private/var/lib/apt`, `/Library/MobileSubstrate/MobileSubstrate.dylib`), `fork()`/`vfork()`/`system()`/`popen()` succeeding (sandbox should block them), `canOpenURL:` on `cydia://`/`sileo://`/`zbra://`, a sandbox-violation write test (`Data.write` to `/private/...` outside the container), `fstat`/`stat` on suspicious paths, and `_dyld_image_count`/`_dyld_get_image_name` enumeration for injected images. **Under R, absence of any such check is the finding; presence is the offensive map of what to hook.** (`MASVS-RESILIENCE-1`, MASWE-0097 "Root/Jailbreak Detection Not Implemented", static **atomic `MASTG-TEST-0240`** "Jailbreak Detection in Code"; runtime counterpart `MASTG-TEST-0241` "Runtime Use of Jailbreak Detection Techniques"; legacy `MASTG-TEST-0088` "Testing Jailbreak Detection" — **deprecated**, superseded by -0240/-0241; knowledge `MASTG-KNOW-0084`)
- **Robustness, not just presence (static → runtime)** — a single hardcoded path check, an `if (jailbroken) {…}` with the detection result stored in one BOOL, or detection living in a never-called function is effectively no detection. Statically note single-point-of-failure and whether the routine appears reachable from launch/security paths; **mark effectiveness Needs dynamic verification** (the bypass is confirmed only on a jailbroken device + Frida). (`MASVS-RESILIENCE-1`, MASWE-0097, `MASTG-TEST-0241` (runtime))
- **Device attestation (R, presence only)** — references to App Attest / DeviceCheck (`DCAppAttestService`, `attestKey`, `generateAssertion`, `DCDevice`) that bind app/device integrity server-side. Statically you can only note presence/absence; server-side enforcement is out of static scope. (`MASVS-RESILIENCE-1`, MASWE-0100 "Device Attestation Not Implemented"; descriptive test name "Device/app attestation present" (ID to confirm) — no clean atomic iOS test, cite control + name)

### RESILIENCE-4 — anti-dynamic-analysis (anti-debug, anti-hook, emulator, debuggable)
- **Anti-debugging — `ptrace(PT_DENY_ATTACH)`** — references to `ptrace` (often resolved via `dlsym`/`syscall(SYS_ptrace,…)` to dodge a direct import) with request `PT_DENY_ATTACH` (31), which makes the kernel kill the process on debugger attach. Also `getppid()` comparisons. Static finds the call; whether it runs early and on every relevant path is runtime. (`MASVS-RESILIENCE-4`, MASWE-0101 "Debugger Detection Not Implemented" (site also titles this "Anti-Debugging"), static refs via "Anti-Debugging Detection" `MASTG-TEST-0089` — **legacy umbrella**, atomic static successor to confirm; knowledge `MASTG-KNOW-0085` "Anti-Debugging Detection")
- **Anti-debugging — `sysctl`/`getppid` debugger check** — the `sysctl` `KERN_PROC` query (`CTL_KERN, KERN_PROC, KERN_PROC_PID, getpid()`) inspecting `kp_proc.p_flag & P_TRACED` to detect an attached debugger; symbol refs to `sysctl`/`sysctlbyname`. *(Fold with the `ptrace` bullet into one anti-debug finding — they are the same control, different primitive.)* (`MASVS-RESILIENCE-4`, MASWE-0101, `MASTG-TEST-0089` (legacy); atomic successor to confirm)
- **Anti-hook / Frida / Substrate / Cydia-Substrate detection** — refs that detect injected instrumentation: `MSHookFunction`/`MSFindSymbol` (Cydia Substrate), `fishhook`/`rebind_symbols`, `frida`/`gum`/`gadget`/`frida-agent` strings, default Frida ports/pipe (`27042`), scanning `_dyld_image_count`/`dladdr` for `FridaGadget`/`SubstrateLoader.dylib`/`libcycript`, `dyld_get_image_name` substring matches, and inline-hook integrity checks on `objc_msgSend`/critical function prologues. **Under R, absence is the finding; presence tells you exactly which artifacts to rename/hide when bypassing.** (`MASVS-RESILIENCE-4`, MASWE-0102 "Dynamic Analysis Tools Detection Not Implemented"; no atomic iOS test yet — descriptive test name "Hook/instrumentation detection present" (ID to confirm), cite control + name)
- **Emulator / Simulator / virtualization detection** — refs to `TARGET_OS_SIMULATOR`/`TARGET_IPHONE_SIMULATOR` compile guards left as runtime checks, env vars `SIMULATOR_DEVICE_NAME`/`SIMULATOR_ROOT`/`DYLD_ROOT_PATH`, `sysctlbyname("hw.model"/"hw.machine")` against `x86_64`/`arm64` mismatch on a "device" build, and app-virtualization/dual-instance markers. Note iOS rarely runs in third-party VMs, so this is lower-yield than on Android — still an R control. (`MASVS-RESILIENCE-4`/`-2`, MASWE-0099 "Emulator Detection Not Implemented" (+ MASWE-0098 "App Virtualization Environment Detection Not Implemented" for the virtualization variant), runtime **atomic `MASTG-TEST-0367`** "Runtime Use of Virtual Device Detection Techniques"; static-refs successor to confirm)
- **`get-task-allow` / debuggable on a release build** — read the embedded code-signature entitlements (`codesign -d --entitlements :- <app>` or `ldid -e`): `get-task-allow=true` lets a debugger attach to the *shipped* app — a genuine RESILIENCE finding on a release artifact (development/ad-hoc builds set it by design, so first confirm this is a RELEASE build per the Workflow). Also flag the `com.apple.security.get-task-allow` entitlement present in `entitlements.plist`. (`MASVS-RESILIENCE-4`, MASWE-0067 "Debuggable Flag Not Disabled", **atomic `MASTG-TEST-0261`** "Debuggable Entitlement Enabled in the entitlements.plist"; legacy `MASTG-TEST-0082` — superseded by -0261)

### RESILIENCE-2 — anti-tampering / integrity self-checks
- **File-integrity / code-signature self-check** — refs that verify the app hasn't been repackaged/patched: reading the embedded `_CodeSignature/CodeResources` and re-hashing bundle resources, `SecCodeCopySelf`/`SecStaticCodeCheckValidity`/`SecCodeCheckValidity` (verify the running code's signature), `csops`/`CS_OPS_STATUS` (`SecTask`), Mach-O `LC_CODE_SIGNATURE`/`__TEXT` checksum comparison, or detecting an added `SC_Info`/re-signed embedded provisioning profile. Absence under R = a repackaged/patched IPA runs undetected. Static finds the mechanism; it cannot confirm it actually rejects a tampered binary → **Needs dynamic verification**. (`MASVS-RESILIENCE-2`, MASWE-0104 "App Integrity Not Verified" (+ MASWE-0105 "Integrity of App Resources Not Verified" for resource hashing; MASWE-0107 "Runtime Code Integrity Not Verified" for in-memory/inline-patch detection — runtime **atomic `MASTG-TEST-0354`** "Runtime Use of Hook Detection Techniques"), legacy `MASTG-TEST-0090` "Testing File Integrity Checks"; atomic successor to confirm)
- **Store-origin / receipt validation** — refs to App Store receipt validation (`appStoreReceiptURL`, `Bundle.main.appStoreReceiptURL`, on-device receipt parsing, or StoreKit2 `Transaction`/`AppTransaction` verification) confirming the binary came from the App Store, not a sideload. Static = presence/absence only. (`MASVS-RESILIENCE-2`, MASWE-0106 "Official Store Verification Not Implemented"; descriptive test name "Store/receipt origin verification" (ID to confirm))
- **Outdated code-signature format** — legacy/v1 signature, or missing hardened-runtime / library-validation flags in the signature, weakening tamper resistance. *(Low priority — one bullet; do not expand.)* (`MASVS-RESILIENCE-2`, MASWE-0104 "App Integrity Not Verified", **atomic `MASTG-TEST-0220`** "Usage of Outdated Code Signature Format"; legacy `MASTG-TEST-0081`)

### RESILIENCE-3 — anti-static-analysis (obfuscation, symbol stripping)
- **Obfuscation absent / logic & strings fully readable** — is the binary meaningfully obfuscated, or do `class-dump`/`nm`/`strings`/Ghidra recover clear ObjC/Swift class & method names, plaintext detection thresholds, endpoint strings, and straight-line control flow? **Caveat:** Swift demangling (`swift-demangle`, `__swift5_typeref`/`__swift5_reflstr` sections) often yields readable type/method names even without ObjC — readable Swift metadata is normal and not itself "no obfuscation"; judge by whether *security-relevant* logic/strings are protected. Required only under R. (`MASVS-RESILIENCE-3`, MASWE-0089 "Code Obfuscation Not Implemented" (+ MASWE-0090 "Resource Obfuscation Not Implemented", MASWE-0091 "Anti-Deobfuscation Techniques Not Implemented"), legacy `MASTG-TEST-0093` "Testing Obfuscation"; atomic successor to confirm)
- **Debug symbols not stripped** — release Mach-O still carries a rich symbol table (`nm -a` shows local symbols), embedded `.dSYM`/DWARF, or `__swift5_*` reflection metadata that eases reversing. (`MASVS-RESILIENCE-3`, MASWE-0093 "Debugging Symbols Not Removed", **atomic `MASTG-TEST-0219`** "Testing for Debugging Symbols"; legacy `MASTG-TEST-0083`)
- **Non-prod / control-disabling code compiled into release** — `#if DEBUG` blocks shipped, staging/test hosts, hidden flags that disable pinning/jailbreak/anti-debug checks, swizzling/test toggles, verbose `NSLog`/`os_log`/`print` of class names, stack traces, or endpoints that aid reverse engineering (distinct from STORAGE-2 *secret* leakage — here it is RE-assistance). *(One bullet; if the same logging also leaks secrets, cross-ref MASVS-STORAGE-2, don't duplicate.)* (`MASVS-RESILIENCE-3`, MASWE-0094 "Non-Production Resources Not Removed" / MASWE-0095 "Code That Disables Security Controls Not Removed", RE-aiding logging **atomic `MASTG-TEST-0358`/`-0359`** (MASWE-0094); legacy `MASTG-TEST-0084`)

## Classic false positives
Consult before promoting a candidate. Each below is **usually NOT a finding** absent extra evidence:

| Candidate | Why it's usually a false positive | When it *is* real |
|---|---|---|
| "No jailbreak / anti-debug / anti-hook / obfuscation" | Under **L1/L2 these are not required at all** — RESILIENCE is assessed **only under R**; reporting their absence outside R is a mis-scope | the engagement is **R** (anti-fraud / DRM / paywalled) and the control is genuinely absent |
| "No detection refs found in the binary" | Binary is **FairPlay-encrypted** (`cryptid=1`) → tool output is ciphertext, your read is invalid | binary is decrypted (`cryptid=0`) and the refs are genuinely absent |
| `get-task-allow=true` / debuggable entitlement | Expected in a **development/ad-hoc** build | present in the **shipped/release** artifact (RESILIENCE-4) |
| Readable Swift type/method names via demangling | Swift reflection metadata (`__swift5_typeref`) is readable by design; not "obfuscation absent" on its own | security-relevant logic/strings/thresholds are plainly readable under R |
| Jailbreak/anti-debug routine present | Don't claim it *works* from static alone — single-point checks are trivially patched | dynamic test on a jailbroken device confirms it fires and resists bypass (or confirms it's dead/bypassable) |
| `TARGET_OS_SIMULATOR` guard in the binary | Normal compile-time guard, not a runtime emulator-detection control | it's an actual runtime check gating security behavior and required under R |
| "No PIE / no stack canary" treated as RESILIENCE | Those are **MASVS-CODE-4 / MASWE-0116** compiler-hardening, not RESILIENCE | — (re-anchor to CODE; see `references/ios/code.md`) |
| Symbol stripping flagged as anti-tampering | Stripping resists **static** analysis → RESILIENCE-**3**, not -2 | — (re-anchor the number within the right group) |

When you dismiss a candidate, record it (and the reason) in the report's "candidates not promoted" appendix.
